# Responsible Use — NanoFinance 负责任使用指南

## 概述

本文档描述 NanoFinance 项目（模型 + Tokenizer + RAG 系统）的负责任使用指南，包括适用场景、不适用场景、风险缓解措施、使用者责任与禁止用途。本项目为研究原型，约 1.4B 参数，配套 RAG 检索增强系统，**不承担金融决策责任**，**不声称完全消除幻觉**，**不声称支持原生 Function Calling**。使用者应始终对模型输出进行人工审核，并遵守本指南的禁止用途规定。

---

## 1. 适用场景

| 适用场景 | 说明 | 前提条件 |
| --- | --- | --- |
| 中文金融报告问答 | 在 RAG 系统提供的证据上下文中回答问题 | RAG 索引已建立，Answerability 校验通过 |
| 金融文本摘要 | 对金融报告段落、会议纪要进行结构化摘要 | 输入文本来自可信来源 |
| 信息抽取 | 从金融文本中抽取关系、情感、实体 | 输出经人工审核 |
| RAG 有证据回答 | 基于 RAG 检索证据生成带来源的回答 | Grounding 校验通过 |
| 确定性计算辅助 | 配合 RAG Calculator 完成 9 种金融数值计算 | 计算由 Calculator 执行，非模型推理 |
| 研究与原型开发 | 学术研究、内部原型、能力探索 | 不用于生产环境金融决策 |

> 所有适用场景均建议在 RAG 系统 + 人工审核流程中使用，**不建议**直接使用裸模型进行金融任务。

---

## 2. 不适用场景

| 不适用场景 | 原因 |
| --- | --- |
| 投资建议 | 模型不提供投资建议，不承担金融决策责任 |
| 证券买卖决策 | 不可作为证券交易决策依据 |
| 自动交易 | 不可接入自动交易系统执行真实交易 |
| 税务法律意见 | 不提供税务、法律专业意见 |
| 实时市场预测 | 模型无实时数据接入，不具备市场预测能力 |
| 无来源计算 | 在没有 RAG 证据或 Calculator 的情况下，模型自行数值计算不可信 |
| 医疗信贷决策 | 不可用于医疗、信贷、保险等高风险个人决策 |
| 生产环境金融决策 | 模型为研究原型，不建议直接用于生产环境金融决策 |

---

## 3. 风险缓解措施

| 风险 | 缓解措施 |
| --- | --- |
| 金融决策风险 | 明确声明不承担金融决策责任；禁止投资建议用途；要求人工审核 |
| 幻觉风险 | RAG 三层校验（Answerability + Grounding + Validation）+ Safe Fallback；人工审核 |
| 时效性风险 | 明确声明无实时数据；标注训练数据截止时间 |
| 索引质量风险 | Answerability 校验；Safe Fallback；索引质量监控 |
| 评测误用风险 | 明确声明 0/54 不代表质量；禁止作为质量指标引用 |
| Tool Calling 误用风险 | 明确声明未训练原生 Tool Calling |
| 数据许可风险 | 各数据源许可待确认，不声明可自由再分发 |
| PII 风险 | 未经过独立 PII 审计，建议在受控环境使用 |

---

## 4. 使用者责任

| 责任 | 说明 |
| --- | --- |
| 人工审核 | 使用者必须对模型输出进行人工审核，不可直接采纳 |
| 决策责任 | 基于模型输出的所有决策由使用者自行承担责任 |
| 合规审查 | 使用者需确保使用方式符合当地法律法规 |
| 数据许可核实 | 使用者需自行核实各数据源的许可条款 |
| 索引质量保障 | 使用 RAG 系统时，使用者需保障索引质量与时效性 |
| 输出标注 | 对外展示模型输出时，应标注为模型生成，并说明局限性 |
| 评测结果表述 | 引用 Phase 5 评测结果时，必须说明 0/54 不代表质量 |
| 不虚构能力 | 不得声称模型具备原生 Function Calling、完全消除幻觉等未经验证的能力 |

---

## 5. 禁止用途

以下用途**明确禁止**：

| 禁止用途 | 说明 |
| --- | --- |
| 投资建议 | 不得将模型输出作为投资建议提供给第三方 |
| 证券买卖 | 不得将模型输出作为证券买卖决策依据 |
| 自动交易 | 不得将模型接入自动交易系统执行真实交易 |
| 税务法律意见 | 不得将模型输出作为税务、法律专业意见 |
| 实时市场预测 | 不得将模型用于实时市场预测 |
| 医疗信贷决策 | 不得将模型用于医疗、信贷、保险等高风险个人决策 |
| 任意 Python 执行 | 不得将模型用于执行任意 Python 代码 |
| 虚假宣传 | 不得声称模型具备原生 Function Calling、完全消除幻觉等未经验证的能力 |
| 质量指标误用 | 不得将 Phase 5 的 0/54 strict pass 作为质量指标引用 |
| 无授权再分发 | 在数据源许可确认前，不得再分发训练数据 |

## 6. 对外表述规范

为避免误导，对外表述模型时应遵循以下规范：

| 表述 | 是否允许 | 说明 |
| --- | --- | --- |
| "约 1.4B 参数的中文金融领域大模型" | ✅ 允许 | 符合事实 |
| "基于 karpathy/nanochat (MIT License)" | ✅ 允许 | 符合事实 |
| "配套 RAG 检索增强系统" | ✅ 允许 | 符合事实 |
| "RAG 系统包含确定性 Calculator" | ✅ 允许 | 符合事实 |
| "模型具备原生 Function Calling" | ❌ 禁止 | 未训练原生 Tool Calling |
| "完全消除幻觉" | ❌ 禁止 | 不能保证消除幻觉 |
| "Tokenizer 带来推理倍速" | ❌ 禁止 | 未经验证 |
| "Phase 5 评测 0/54 strict pass" | ⚠️ 限制 | 仅可说明为基础设施功能测试，不可作为质量指标 |
| "引入 R1 风格的显式推理格式对齐数据" | ✅ 允许 | 正确表述 |
| "改造成 R1 架构" | ❌ 禁止 | 模型架构未改变 |
| "模型数学计算精度由 Calculator 保证" | ✅ 允许 | Calculator 为确定性算法 |
| "模型自身数学推理能力等同于 Calculator" | ❌ 禁止 | Calculator 正确性非模型能力 |

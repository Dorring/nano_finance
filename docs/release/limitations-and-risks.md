# Limitations and Risks — NanoFinance 局限性与风险

## 概述

本文档汇总 NanoFinance 项目（模型 + Tokenizer + RAG 系统）的局限性与风险。本项目是一个基于 karpathy/nanochat 的中文金融领域大模型研究原型，约 1.4B 参数，配套 RAG 检索增强系统。本卡片明确声明：模型规模有限、金融知识有时效性、不能保证消除幻觉、无原生 Tool Calling、RAG 依赖索引质量、不承担金融决策责任。Phase 5 的 synthetic-held-out 0/54 strict pass **不代表质量**，不作为任何质量指标引用。

---

## 1. 模型规模限制

| 局限性 | 说明 | 来源 |
| --- | --- | --- |
| 参数量 | 约 1.4B | [已验证] |
| 与主流模型对比 | 小于主流金融大模型（7B+） | 行业常识 |
| 影响 | 复杂推理、长链推理、多步逻辑能力有限 | — |

> 约 1.4B 参数的模型在复杂金融推理任务上能力有限，不应期望其达到 7B+ 模型的表现。

---

## 2. 金融知识时效性

| 局限性 | 说明 | 来源 |
| --- | --- | --- |
| 训练数据截止 | 训练数据有截止日期 | [已验证] |
| 最新市场动态 | 不包含训练截止后的市场动态 | [已验证] |
| 法规变更 | 不包含训练截止后的法规变更 | [已验证] |

> 模型的金融知识停留在训练数据截止时间，**不可用于**获取最新市场信息或法规变更。

---

## 3. 训练语料偏差

| 局限性 | 说明 | 来源 |
| --- | --- | --- |
| 中文金融语料占比 | 8.54%（预训练） | [历史自报] |
| 数据来源偏差 | ClimbMix、SkyPile、中文金融数据各有来源偏差 | [已验证] |
| 领域偏差 | 金融数据可能集中于特定子领域（如财报） | [待验证] |

> 训练语料的组成偏差可能导致模型在某些金融子领域表现较好，而在其他子领域表现较差。

---

## 4. 英文能力与中文能力差异

| 局限性 | 说明 | 来源 |
| --- | --- | --- |
| 主要目标语言 | 中文 | [已验证] |
| 英文能力 | 依赖 ClimbMix 训练数据 | [已验证] |
| 金融专业英文 | 可能弱于中文金融表现 | [待验证] |

> 模型以中文金融为主要目标，英文金融专业表现可能弱于中文。

---

## 5. 复杂跨表计算限制

| 局限性 | 说明 | 来源 |
| --- | --- | --- |
| 跨表计算 | 复杂跨表、多步数值计算由 RAG Calculator 处理 | [已验证] |
| 模型自身能力 | 模型自身跨表计算能力有限 | [已验证] |
| Calculator 范围 | Calculator 仅支持 9 种操作 | [已验证] |

> 复杂跨表计算依赖 RAG Calculator 的 9 种确定性操作，超出这 9 种操作的计算需求无法保证正确性。

---

## 6. 无原生 Tool Calling 训练

| 局限性 | 说明 | 来源 |
| --- | --- | --- |
| 原生 Function Calling | **未训练** | [已验证] |
| Tool Calling | **未训练** | [已验证] |
| 重要声明 | **不声称支持原生 Function Calling / Tool Calling** | [已验证] |

> 模型未训练原生 Function Calling / Tool Calling 能力。RAG 系统的 Calculator 等工具调用由系统编排，**不是**模型的原生能力。

---

## 7. RAG 依赖索引质量

| 局限性 | 说明 | 来源 |
| --- | --- | --- |
| 检索质量 | 受 ChromaDB/BM25 索引质量影响 | [已验证] |
| 索引缺失 | 索引中缺失的文档无法被检索 | [已验证] |
| 索引错误 | 索引中错误的文档会导致回答失败 | [已验证] |

> RAG 系统的回答质量上限受索引质量约束。索引缺失或错误会导致 Answerability 校验失败或回答质量下降。

---

## 8. Validation 不能验证所有自然语言事实

| 局限性 | 说明 | 来源 |
| --- | --- | --- |
| Validation 范围 | 规则/校验系统，不能验证所有自然语言事实 | [已验证] |
| fail-closed | 校验失败时阻断回答 | [已验证] |
| 重要声明 | **不能保证消除幻觉** | [已验证] |

> Validation 为 fail-closed 设计，但不能验证所有自然语言事实。即使通过 Validation，模型仍可能产生幻觉。**不声称完全消除幻觉**。

---

## 9. 不能保证消除幻觉

| 局限性 | 说明 | 来源 |
| --- | --- | --- |
| 幻觉风险 | 模型仍可能产生幻觉 | [已验证] |
| RAG 缓解 | RAG 通过 Grounding 校验缓解幻觉，但无法完全消除 | [已验证] |
| 重要声明 | **不声称完全消除幻觉** | [已验证] |

> 即使在 RAG 系统的 Answerability + Grounding + Validation 三层校验下，模型仍可能产生幻觉。使用者应始终对模型输出进行人工审核。

---

## 10. 无实时数据

| 局限性 | 说明 | 来源 |
| --- | --- | --- |

| 实时检索 | RAG 依赖预先建立的索引 | [已验证] |

> 模型与 RAG 系统均无实时市场数据接入，**不可用于**实时市场预测或实时信息获取。

| 局限性 | 说明 | 来源 |
| --- | --- | --- |
| Phase 5 数据分类 | `synthetic_held_out` | [已验证] |
| 是否真正独立 Sealed Evaluation | **否** | [已验证] |
| 0/54 strict pass | **不代表质量** | [已验证] |
| 禁止用途 | 不作为质量估计、模型比较、简历指标 | [已验证] |

> **核心声明**：Phase 5 的 0/54 strict pass 基于 `synthetic_held_out` 数据分类，**不是真正独立 Sealed Evaluation**。该结果**仅用于基础设施功能测试**，**不代表**模型质量，**不可**作为质量估计、模型比较或简历指标引用。详见 `evaluation-card.md`。

## 13. 风险汇总

| 风险类别 | 风险描述 | 缓解措施 |
| --- | --- | --- |
| 金融决策风险 | 使用者可能将模型输出作为投资/交易决策依据 | 明确声明不承担金融决策责任，禁止投资建议用途 |
| 幻觉风险 | 模型可能产生不符合事实的回答 | RAG 三层校验 + Safe Fallback + 人工审核 |
| 时效性风险 | 使用者可能误以为模型包含最新信息 | 明确声明无实时数据，知识有截止日期 |
| 索引质量风险 | 索引缺失/错误导致回答失败 | Answerability 校验 + Safe Fallback |
| 评测误用风险 | 0/54 被误用为质量指标 | 明确声明 0/54 不代表质量，禁止作为质量指标引用 |
| Tool Calling 误用风险 | 使用者误以为模型支持原生 Function Calling | 明确声明未训练原生 Tool Calling |
